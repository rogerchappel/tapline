# Roadmap

## Now

- Keep the MVP local-first and read-only by default.
- Improve formula parsing only when backed by fixtures.
- Make release review output easy to paste into PRs and GitHub releases.

## Next

- Add policy configuration for required livecheck/test blocks.
- Add more fixture tap layouts.
- Add GitHub Actions annotations or SARIF output.
- Document recommended Homebrew validation sequences by release scenario.

## Later

- Consider optional integration with release tooling.
- Consider changelog-aware release notes.
- Consider package distribution after real maintainer feedback.

## Not planned

- Automatic publishing.
- Credential discovery.
- Hidden network checks.
- Replacing Homebrew's own audit/test commands.
